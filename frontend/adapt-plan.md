# AUTO-EVO-AI 适配非技术人员 & 小微企业方案

> 基于当前系统能力，分析适配路径和优先级
> 日期：2026-06-30

---

## 一、核心矛盾

AUTO-EVO 的底层架构是为"技术创业者"设计的。适配非技术用户不是加几个按钮的事，是需要**重新定义交互方式**。

但好消息是：**85% 的后端能力不需要改**，只需要换一层皮和补一些基础设施。

---

## 二、非技术人员适配方案

### 当前阻碍非技术用户的门槛

| 门槛 | 当前状态 | 改造成本 |
|---|---|---|
| 部署 | SSH + Docker + API Key 配置 | 高（但一次性的） |
| 登录 | 要填用户名/密码/API Key | **低** |
| 入门 | 583 个模块铺在那里，无从下手 | **中** |
| 对话 | 要自己输入指令，不知道能干什么 | **低** |
| 反馈 | 没有引导，自己试错 | **中** |
| 前端 | 按钮点不动，体验差 | **低**（已在修） |

### 改造路线

#### P0：注册后引导（2-3 天）

```
首次登录 → 弹窗问 3 个问题：
1. 你是做什么的？（创业者/自由职业/小团队/个人用）
2. 你主要想用 AI 做什么？（写文档/数据分析/自动化流程/客服/编程）
3. 你用什么工具？（微信/飞书/钉钉/邮件/都不重要）

→ 自动配置建议的系统预设
→ 自动打开对应的工具分类
```

前端工作量：弹窗 + 3 个选择题 + 预设配置。等价于写一个表单页面。

#### P1：预设模板（1 天）

预置 10 个场景模板，每种回答 3 个问题后系统自动激活：

```
1. 「我是小老板，管 3 个人」 → 任务分配 + 审批流 + 通知
2. 「我写公众号」 → AI 写作 + 图片生成 + 定时发布
3. 「我做电商」 → 商品描述 + 客服自动回复 + 数据分析
4. 「我开了个小公司」 → 合同审查 + 发票管理 + 邮箱通知
5. 「我是设计师」 → 图片生成 + 代码转设计 + 截图转代码
```

每个模板其实就是**预设的对话提示词 + 打开对应工具分类**，不涉及后端修改。

#### P2：对话式入口（3-5 天）

当前：用户要在输入框里打字 `帮我写一份合同`。
问题：用户不知道可以说什么。

改造方案：
```
首次打开对话页面时，显示 6 个推荐卡片：
「写合同」「做表格」「查资料」「画流程图」「生成图片」「自动回复客户」
点击任意卡片 → 自动填入提示词 → 用户按发送就行
```

这等价于在聊天框上方加一排快捷入口按钮，不需要改后端。

#### P3：免配置部署（1-2 周）

这是最大的门槛。当前需要：
1. 购买服务器
2. SSH 连接
3. Docker 安装
4. API Key 配置
5. 域名解析

对非技术用户 = 不可能。

改造方案：
```
A. 提供 Docker Compose 一键部署 → docker-compose up
B. 云市场镜像 → 腾讯云/阿里云 买服务器直接预装
C. 提供 SaaS 托管版 → 用户注册即用（但涉及钱了）
```

## 三、小微企业适配方案

### 小微企业的核心需求

| 需求 | 当前状态 | 改造 |
|---|---|---|
| 多用户 | 无（单用户） | **需新功能** |
| 角色权限 | 无（admin 全部） | **需新功能** |
| 数据隔离 | 无 | **需新功能** |
| 仪表盘 | 有（enterprise.html） | 已有 |
| 通知 | 有（13 通道） | 已有 ✅ 最强功能 |
| 自动化 | 有（循环/工作流/调度） | 已有 |

### 最低 MVP（最小可行产品）

不需要多租户，不需要角色权限。小微企业 2-5 个人，**共享一个账号也能用**。

只需要做一件事：**安装向导**。

```
小微企业版安装流程：

1. 下载 → docker-compose.yml
2. 运行 → docker-compose up -d  
3. 浏览器打开 → 自动进入设置向导
   a. 选用途（创业/自由职业/小团队）
   b. 填通知方式（微信/邮件/钉钉）
   c. 点完成
4. 进入系统 → 已经预配好推荐模板
```

这些东西后端能力全部就绪，缺的是**引导层**。

### 商业化方向

如果要做真正的企业版，需要的功能：

| 功能 | 工作量 | 备注 |
|---|---|---|
| 注册+登录（邮箱） | 3-5 天 | 现有登录页改造 |
| 多用户+角色 | 1-2 周 | 后端架构调整 |
| 数据隔离 | 1 周 | 每个用户独立空间 |
| 用量配额 | 已有（桩模块） | 填真实逻辑 |
| 付费墙 | 已有（桩模块） | 填真实逻辑 |
| 用户管理后台 | 1 周 | admin.html 扩展 |

## 四、具体实施路线

### Phase 1（1 周）— 让非技术人员能用

目标：一个第一次接触的用户能在 5 分钟内开始使用

1. ✅ 修复前端按钮（完成中）
2. 简化登录页（去 API Key 字段）
3. 添加首次使用引导（3 个问题 → 预设）
4. 添加对话快捷入口（6 个推荐卡片）

### Phase 2（2 周）— 让小微企业用起来

目标：团队 2-5 人可以共享协作

1. 多用户登录（共享 admin）
2. 通知配置向导
3. 仪表盘增加团队协作视图
4. 操作审计日志

### Phase 3（1 个月）— 商业化

1. 注册登录 + 权限系统
2. 用量配额
3. 付费订阅
4. 云托管版

## 五、结论

**能不能做？能。但前提是把按钮先全部修好。**

非技术用户对前端 bug 的容忍度是 0——第一印象就是全部。你作为一个开发者可以忍，但一个不会写代码的人第一次看到按钮点不动就直接关网页了。

**后端能力已经远超小微企业的需求。** 13 通道通知、工作流引擎、AI 对话、模块热加载——小微企业需要的东西 90% 都有了。缺的是：

1. 前端体验（按钮能点）
2. 引导流程（装好就能用）
3. 多用户（共享协作）

这三个的改造成本对后端架构来说是轻微的。但**第 1 条不做完，2 和 3 都没有意义**。
