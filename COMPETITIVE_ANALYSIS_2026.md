## 一、2026 格局：三家独大

2026 年视觉 AI Agent 构建工具已经形成三足鼎立，**每家定位截然不同**：

| 项目 | 定位 | Stars | 核心哲学 | 我们 vs 它 |
|------|------|-------|---------|-----------|
| **Dify** | LLMOps 平台 | **142k** ⭐ | 端到端应用交付 | 🟡 功能对齐，缺可视化Workflow |
| **n8n** | 通用自动化引擎 | **130k+** ⭐ | 事件驱动+海量集成 | 🟢 AI更强，缺 5000+ 连接器 |
| **Langflow** | LLM 图形构建器 | 60k+ ⭐ | 透明的 LangChain 数据流 | 🟢 模块覆盖更广 |
| **Flowise** | 低代码 Chatflow | 70k+ ⭐ | 极简 RAG 构建 | 🟢 多模态更强 |

**关键结论：** 2026 年最佳实践不是选一个，而是**组合使用**——MCP 使它们成为互操作的组件。

## 二、深度解析：三家核心值得抄的

### 1. Dify (142k⭐) — 端到端 LLMOps 平台

**Dify 最强的地方：**

| 能力 | Dify 实现 | AUTO-EVO-AI 现状 | 差距 |
|------|----------|-----------------|------|
| **"知识"功能** | 上传→分块→嵌入→混合搜索→重排序，全程可配置 | 🟡 有 Rerank API，无前端流程 | 🚨 **最值得抄** |
| **应用封装** | 每个 Chatflow 自动暴露 REST API + 托管聊天 UI | ✅ 我们有 API 和聊天 UI | 🟢 |
| **用户/日志/评估** | 内置用户管理、日志、Prompt 版本管理、人工标注评估 | 🟡 有 JWT + 日志，无版本评估 | 🟡 |
| **MCP 集成** | 支持接入外部 MCP 工具 | ✅ 自动发现+桥接 Skills | 🟢 |
| **插件系统** | v1.x 插件市场，第三方贡献 | ✅ 插件商店路由 | 🟢 |

**最值得借鉴的 Dify 功能：**
```
1. "知识"端到端 RAG 流程：上传文档→AI 分块→向量化→检索配置→重排序→LLM 回复
2. Prompt 版本管理：每次修改保存版本，可回滚对比
3. 应用模板市场：预置 100+ 行业模板，一键部署
```

### 2. n8n (130k+⭐) — 通用自动化引擎

**n8n 最强的地方：**

| 能力 | n8n 实现 | AUTO-EVO-AI 现状 | 差距 |
|------|----------|-----------------|------|
| **5000+ 连接器** | 社区贡献的 npm 包生态 | ✅ 20 内置连接器 | 🔴 **巨大差距** |
| **事件触发** | 200+ 触发类型（Webhook/DB变更/邮件/定时） | 🟡 Webhook + 定时调度 | 🟡 |
| **错误处理分支** | 节点失败自动走错误路径，可重试/回退 | 🟡 基础 try-catch | 🟡 |
| **AI Agent 节点** | Agent 工具可以是任意 n8n 节点 | 🟢 原生 AI 路由 | 🟢 |
| **执行历史** | 每次执行完整记录，可回放 debug | 🟡 基础日志 | 🟡 |

**最值得借鉴的 n8n 功能：**
```
1. 连接器架构：每个连接器一个独立 npm 包，`npm install n8n-nodes-xxx` 即装即用
2. 错误处理分支：节点旁自动出现"success/failure"两条路径
3. 执行日志回放：点开历史执行记录，逐节点展开输入/输出
```

### 3. Langflow (60k+⭐) — LLM 图形构建器

**Langflow 最强的地方：**

| 能力 | Langflow 实现 | AUTO-EVO-AI 现状 | 差距 |
|------|--------------|-----------------|------|
| **组件体系** | 每个节点 = 一个 LangChain 概念（Prompt/LLM/Retriever/Agent） | 🟢 我们有 483 模块 | 🟢 |
| **数据流可视化** | 节点连线，边显示类型化的对象流 | ❌ 无可视化 | 🔴 |
| **MCP 互操作** | 可做 MCP Server，也可消费 MCP 工具 | ✅ 双向 | 🟢 |
| **Flow→API 一键导出** | 每个 Flow 自动变 REST 端点 | ✅ 已有 API | 🟢 |

**最值得借鉴的 Langflow 功能：**
```
1. 节点类型化：边传输类型化对象（str/list/embedding/document），让数据流可见
2. Flow 导出 JSON：拖拽完导出可版本化的 JSON 配置
```
