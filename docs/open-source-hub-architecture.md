# AUTO-EVO-AI 开源中心 · 架构设计

> 让**任何人**都能**可视化地组装开源世界**
>
> 核心能力只有五个：
> - 🧩 **增加** — 发现并加入任何开源项目
> - ✂️ **删除** — 不需要的一键移除
> - 🔗 **组合** — N个项目 → 1个新项目
> - ✏️ **修改** — 改配置/改代码/改界面
> - 🧬 **二次开发** — 在开源项目基础上衍生自己的项目
>
> 最终：**普通用户像搭积木一样，把开源项目拼成自己的工具箱**

---

## 一、产品定位

**开源中心**是 AUTO-EVO-AI 的核心功能模块，核心理念：

### 四个核心操作

```
┌─────────────────────────────────────────────────────┐
│                   开源中心                            │
│                                                     │
│  🧩 增加                          ✂️ 删除           │
│  ┌─────────────────┐             ┌──────────────┐   │
│  │ 从GitHub/HF发现  │             │ 一键移除      │   │
│  │ 拖入工作区        │             │ 清理依赖      │   │
│  │ 自动分析依赖      │             │ 释放资源      │   │
│  └─────────────────┘             └──────────────┘   │
│                                                     │
│  🔗 组合                          ✏️ 修改           │
│  ┌─────────────────┐             ┌──────────────┐   │
│  │ A + B + C = D    │             │ 改端口/环境变量 │   │
│  │ 项目合并为1个    │             │ 改配置文件      │   │
│  │ 统一管理/启动    │             │ 改前端界面      │   │
│  └─────────────────┘             └──────────────┘   │
└─────────────────────────────────────────────────────┘
```

**产品承诺：拿到本系统的任何用户，不需要写一行代码，就能：**

1. 🧩 **增加**：浏览 GitHub/HuggingFace 热门项目 → 点击"加入"
2. 🔗 **组合**：把 Ollama（推理）+ Open WebUI（界面）+ Dify（工作流）→ 拖拽连成"AI平台"
3. ✏️ **修改**：改端口/改主题/改模型参数 → 所见即所得
4. 🧬 **二次开发**：Fork 任意开源项目 → 改代码 → 变成自己的项目
5. ✂️ **删除**：不想要了 → 右键删除 → 全部清理干净

### 3.2 ✂️ 删除 — 一键移除

| 删除类型 | 效果 | 操作 |
|---------|------|------|
| **停用** | 停止进程，保留配置和数据 | 右键→停用 |
| **卸载** | 停止 + 删除代码和数据 | 右键→卸载 |
| **清理** | 卸载 + 清理依赖/Docker镜像 | 右键→清理 |

**智能删除：**
- 检测是否有其他项目依赖此项目 → 提示用户
- 删除前自动备份配置快照 → 可恢复
- 统计释放的资源（磁盘/内存/端口）→ 展示给用户

### 3.3 🔗 组合 — 多项目集成→新项目

**核心价值：N个开源项目 = 1个新功能**

```
┌─────────────┐   ┌──────────────┐   ┌──────────┐
│  Ollama     │   │  Open WebUI  │   │  Dify    │
│  (推理引擎)  │──▶│  (聊天界面)   │──▶│ (工作流) │
└─────────────┘   └──────────────┘   └──────────┘
         │                                  │
         └──────────────┬───────────────────┘
                        ▼
              ┌─────────────────────┐
              │   「我的AI平台」     │ ← 组合后的新项目
              │   一个入口，全套功能  │
              │   ollama:11434      │
              │   webui:8080        │
              │   dify:3000         │
              └─────────────────────┘
```

**组合方式：**

| 方式 | 说明 | 示例 |
|------|------|------|
| **前后串联** | A输出→B输入 | Ollama+Open WebUI |
| **并联聚合** | 统一入口→多个服务 | 聊天+绘图+搜索→统一面板 |
| **嵌套** | B作为A的插件 | Dify集成Ollama作推理后端 |
| **模板化** | 保存组合为模板 | "AI平台"模板一键部署 |

**组合后的新项目：**
- 有独立的名称/图标/描述
- 统一的管理入口（一个端口访问）
- 所有子项目的监控聚合
- 可以导出为模板分享

### 3.4 ✏️ 修改 — 所见即所得

| 修改类型 | 内容 | 操作方式 |

```
┌─────┬──────────────┬──────────┐
```

**集成流程：**

```

③ 生成集成方案

    ├─ 策略B: 进程管理 (nohup/supervisor)

④ 用户预览配置
    └─ 可编辑端口映射/环境变量/资源限制
⑤ 执行集成
⑥ 添加路由 + 添加到导航菜单
⑦ 健康检查 → 完成
```

## 四、数据模型

```python
class OpenSourceProject:
    id: str                    # 唯一ID (hash)
    name: str                  # 项目名
    source: str                # github / huggingface / gitcode
    repo_url: str              # 仓库地址
    description: str           # 描述
    category: str              # 分类
    tags: list[str]            # 标签
    tech_stack: list[str]      # 技术栈
    stars: int                 # 星标数
    license: str               # 许可证
    readme: str                # README (缓存)
    dependencies: dict         # 依赖分析结果
    integrated: bool           # 是否已集成
    status: str                # 状态 (running/stopped/error)
    config: ProjectConfig      # 运行配置
    created_at: datetime
    updated_at: datetime

class ProjectConfig:
    port: Optional[int]        # 映射端口
    env_vars: dict             # 环境变量
    resources: ResourceLimit   # CPU/内存限制
    routes: list[str]          # Evo路由路径
    auto_start: bool           # 开机自启
    health_check: str          # 健康检查URL/命令
    docker_config: dict        # Docker配置

class IntegrationFlow:
    project_id: str
    steps: list[FlowStep]      # 集成步骤
    status: str                # analyzing/ready/deploying/running/failed
    log: list[str]             # 执行日志
    error: Optional[str]       # 错误信息
```

## 五、API 设计

### 5.1 项目发现

```
GET  /api/v1/hub/discover
    参数: source=github, category=AI, sort=trending, page=1, limit=20
    返回: {projects: [...], total: N}

GET  /api/v1/hub/search
    参数: q=langchain, source=all, page=1
    返回: {projects: [...]}

GET  /api/v1/hub/project/{id}
    返回: {project: {...}, readme: "...", integration_status: "..."}
```

### 5.2 项目集成

```
POST /api/v1/hub/integrate
    参数: {project_id, config: {port, env_vars, ...}}
    返回: {flow_id, estimated_time}

GET  /api/v1/hub/integrate/{flow_id}/status
    返回: {status, progress%, log: [...], error?}

POST /api/v1/hub/{id}/start
POST /api/v1/hub/{id}/stop
POST /api/v1/hub/{id}/restart
DELETE /api/v1/hub/{id}
```

### 5.3 编排

```
POST /api/v1/hub/compose
    参数: {nodes: [...], edges: [...]}
    返回: {compose_id, status}

GET  /api/v1/hub/compose/{id}
    返回: {nodes, edges, status}

POST /api/v1/hub/compose/{id}/deploy
    返回: {deploy_id}
```

### 5.4 监控

```
GET  /api/v1/hub/monitor
    返回: {projects: [{id, cpu, mem, status, uptime}]}

GET  /api/v1/hub/monitor/{id}/logs
    参数: tail=100, search="error"
    返回: {logs: [...]}
```

## 七、部署策略

| 类型 | 方案 | 适用场景 | 复杂度 |
|------|------|---------|--------|
| 🐳 Docker | docker-compose | 有 Dockerfile/Compose | ⭐ |
| 🐍 Python | pip + supervisor | Python项目 | ⭐⭐ |
| 📦 Node | npm + pm2 | JS/TS项目 | ⭐⭐ |
| 📁 二进制 | 下载+chmod+进程管理 | Go/Rust | ⭐⭐ |
| 🔧 源码编译 | git clone + build | 需要编译 | ⭐⭐⭐ |
| 🔌 嵌入式 | Python import | Agent/API模块 | ⭐ |

**自动选择策略：**
1. 检测到 Dockerfile → Docker 部署
2. 仅 requirements.txt → Python venv
3. 仅 package.json → Node + pm2
4. 两者都有 → 优先 Docker
5. 无依赖文件 → 检测语言类型
