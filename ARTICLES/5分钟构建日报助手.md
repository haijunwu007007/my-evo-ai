## 二、日报不是写出来的，是自动生成的

系统预设了 3 个模板任务，覆盖每日核心数据：

| 任务 | 触发器 | 产出 |
|------|--------|------|
| `📊 GitHub Trending 分析` | 每 4 小时 | 25 条趋势项目 |
| `🩺 系统健康巡检` | 每小时 | 全模块状态快照 |
| `🧬 日常自我进化` | 每天 09:00 | 进化报告（趋势+分析） |

访问 Dashboard → 底部"系统洞察"面板，能看到实时运行数据：

```json
GET /api/insights/evolution
{
  "success": false,
  "error": "暂无进化记录"
}
```

为什么是空的？因为进化记录需要**LLM 解读**才产生。配好 Key 后（见下文），每天 9 点系统自动输出进化报告。

## 三、连接 LLM 让系统"活过来"

进化引擎是系统的核心差异。它不是一个定时任务，而是一个闭环：

```
GitHub Trending → Search API 抓取 25 条热门项目
       ↓
规则引擎 → DecisionEngine 分析 10 条预置规则
       ↓
GLM-4 解读 → 对比历史数据，识别趋势
       ↓
进化报告 → 存入数据库，API 可查
       ↓
（循环）→ 24 小时后再次触发
```

配 Key 只需一行：

```bash
# 编辑 .env，填入智谱 API Key
echo "ZHIPU_API_KEY=your_key_here" >> .env
echo "GLM_MODEL=glm-4-plus" >> .env

# 重启服务
python api_server.py
```

之后查看进化报告：

```bash
curl http://127.0.0.1:8765/api/insights/evolution
```

## 四、自动进化的底层实现

拆开来看，"自我进化"不是装概念的营销词，它在代码里有三层实现：

### 第一层：EvoPhase — 进化阶段

```python
from core.evo_brain import EvoPhase, EvoGate

class DailyLearning(EvoPhase):
    def forward(self, trending_repos):
        return self.analyze_patterns(trending_repos)
```

每个进化阶段执行一个具体任务：扫描 → 分析 → 总结 → 存储。

### 第二层：EvoGate — 进化门控

```python
class EvoGate:
    def should_evolve(self, last_report, current_data):
        # 只有数据量足够时才触发进化，防止空跑
        return len(current_data) >= 3
```

EvoGate 控制是否真的执行进化——数据太少时不跑，节省资源。

### 第三层：SkillForge — 技能锻造

```python
class SkillForge:
    def forge_skill(self, topic, complexity):
        # 基于热门技术趋势自动生成新 skill
        return AutoSkill(
            name=f"analyze_{topic}",
            trigger="daily_evolution",
            logic={"action": "analyze", "params": {"focus": topic}}
        )
```

当系统连续 N 天发现某些技术关键词频繁出现（如 "LLM"、"RAG"、"Agent"），SkillForge 自动生成对应的分析模块，注册到系统中。

这 3 个组件组合在一起，实现了 **"发现 → 学习 → 注册 → 遗忘"** 的完整进化周期。
